# GitHub Copilot Prompt: Polkadot AI Mesh Landing Page

Build a production-grade, human-designed landing page for "Polkadot AI Mesh" - a decentralized AI compute and data marketplace for Smart Cities and DeFi on the Polkadot ecosystem. This is a ₹30 crore project proposal requiring exceptional design quality.

## Tech Stack
- Next.js 14+ with TypeScript
- TailwindCSS (already configured)
- Lucide React for icons
- Framer Motion for animations
- React Intersection Observer for scroll animations
- Class Variance Authority (CVA) for component variants

## Design Philosophy - CRITICAL REQUIREMENTS

### ❌ FORBIDDEN Design Elements
- NO violet, pink, or dark blue gradients
- NO typical AI-generated color schemes
- NO generic gradient backgrounds
- NO overly saturated neon colors
- Design must look 0% AI-generated

### ✅ Required Design Approach - DARK THEME (Polkasmesh Brand)
- **Typography**: SF Pro Display, SF Pro Text (macOS system fonts) - use `font-family: -apple-system, BlinkMacSystemFont, 'SF Pro Display', 'SF Pro Text', sans-serif`
- **Color Palette** (Based on Polkasmesh Branding):
  - **Background**: Deep black/navy (#0A0B0D, #0D0E12, #000000)
  - **Primary Accent**: Hot pink/magenta (#FF006E, #E6007A, #FF0080) - for CTAs, highlights, mesh graphics
  - **Text Colors**: 
    - Primary: Pure white (#FFFFFF) for headlines
    - Secondary: Light gray (#A0A0A0, #8E8E93) for body text
    - Muted: Dark gray (#5E5E5E) for subtle text
  - **Mesh/Grid Visualization**: Hot pink (#FF006E) wireframe on dark background
  - **Accent Colors**: 
    - Polkadot pink variants for hover states
    - Subtle purple tints (#8B5CF6) for secondary elements
    - Deep charcoal (#1A1A1A) for card backgrounds
- **Design References**: Apple.com (dark mode), Linear.app, Stripe.com, Vercel.com, Polkadot.network
- **Layout**: Generous whitespace, asymmetric grids, large typography, minimal borders, dark glass morphism effects

## Animation Guidelines
- Use Framer Motion for:
  - Page transitions
  - Scroll-triggered animations (useInView)
  - Hover effects (scale, opacity)
  - Staggered children animations
- Keep animations subtle: 200-400ms duration
- Ease curves: easeInOut for most, easeOut for exits
- Reduce motion for accessibility (prefers-reduced-motion)

## Responsive Breakpoints
- Mobile: 0-640px (single column)
- Tablet: 641-1024px (two columns)
- Desktop: 1025px+ (multi-column, max-width 1440px)

## Performance Requirements
- Lazy load images and heavy components
- Use Next.js Image component with blur placeholders
- Code splitting for route optimization
- Lighthouse score target: 95+ performance, 100 accessibility
